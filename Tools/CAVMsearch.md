# Section 1 - Non-Technical Description

This program accepts one or more search words and looks through a local memory of saved items to find entries that match those words, then prints a ranked list of matching items with how well each match scored, how many saved versions it has, when it was last updated, which of the search words appear in the item and how strongly, a short list of the chain's most important tokens, optional short response excerpts, optional recent version summaries, and optionally related items similar to each result.

# Section 2 - Technical Analysis

The script is a command-line tool that builds a query from the positional keyword arguments and uses a ContextAwareVersionedMemory object from an imported CAVMFunctions module to perform the actual search and token operations.

- Argument parsing: parse_args() reads keywords (one or more) and options including result count (-n), base directory (-b), flags to show versions (-v) and snippets (-s), whether to include dormant chains (--all-chains), an explicit similarity threshold (--threshold), a debug tokens flag, and parameters controlling how many related chains to show and their minimum similarity.

- Initialization and query tokenization: main() constructs a CAVM.ContextAwareVersionedMemory with the provided base directory. It sets the similarity threshold to the provided value or the module's default (cavm.SIMILARITY_THRESHOLD). It concatenates the keywords into a single query string and calls cavm.Tokenize(query_str) to obtain a mapping of tokens to weights (query_ctr). It prints the detected query tokens and chosen threshold and aborts with an error message if tokenization yields no tokens.

- Performing the search: the code calls cavm.Search(query_str, limit=args.number) to obtain a list of results. Each result is expected to be a tuple of (score, cid, vid, top) where score is a numeric similarity, cid is the chain identifier, vid is a version identifier, and top is the chain's top tokens for the matched version. If no results are returned, it prints a message and exits.

- Loading profiles and dormant set: the program loads all stored profiles via cavm.GetProfiles() into profiles. If --all-chains is not set it calls cavm.LoadDormant(args.base) to get a set of dormant chain ids; otherwise it uses an empty set. It filters the search results to exclude dormant chains when --all-chains is not set, and builds candidate_profiles, a dictionary of profiles filtered the same way, for later related-chain comparisons.

- Output header: it prints a 70-character divider and a header line showing the query string.

- Per-result processing: for each returned result (enumerated as rank), it fetches the corresponding profile from profiles. If the profile is missing it skips the entry. It obtains the chain's versions list and selects the latest version (versions[-1]) and its token vector (latest_tokens). It computes token-match statistics by calling compute_token_stats(query_ctr, latest_tokens). That function iterates the latest version's token dictionary (token -> weight), counts tokens whose text exactly matches any token in the query token set, accumulates the matched tokens along with their weights, sums weights for matched tokens, sorts matched tokens by weight descending, and returns (matched_count, total_weight, matched_tokens_list).

- Token totals and overlap: the program computes total_chain_tokens as the sum of all weights in latest_tokens and calculates overlap_pct as total matched weight divided by total_chain_tokens, expressed as a percentage (or 0 if the chain has no tokens).

- Relevance labeling and score formatting: the numeric score from cavm.Search is compared to the threshold to derive a relevance label: "VERY HIGH" if score >= 1.5 * threshold, "High" if >= threshold, "Medium" if >= 0.7 * threshold, else "Low". format_score(score, threshold) returns a string representation of the numeric score, appending a star if score >= 1.5 * threshold, or appending " (below threshold)" if the score is below threshold.

- Printing result fields: for each item the program prints the rank, formatted score, relevance label, chain id, number of versions and latest version number (from latest.get('v')), a localized last-updated time (iso_to_local) plus a relative age computed by age_human (which computes elapsed time between now and a provided datetime with outputs like "5s ago", "3m ago", "2h ago", "4d ago", or "1mo ago"), and the total token count for the latest version. It then prints keyword overlap lines showing how many distinct tokens matched, matched token weight vs total tokens, and the percent overlap. If there are matched tokens, it lists up to eight top matched tokens formatted as token(weight). It prints the chain top tokens passed in the search result (chain_top_tokens) trimmed to ten tokens.

- Optional snippet: if the --show-snippet flag is used and the latest version has a 'response' field, it prints up to the first 180 characters of that response (ellipsis appended if longer), limited to the first two lines of that snippet.

- Optional recent versions: when -v/--versions is set and the chain has more than one version, it iterates up to the last five versions in reverse order. For each version it fetches the version number v.get('v','?'), localizes last_updated via iso_to_local, asks cavm.TopTokens for up to five top tokens from that version's token dictionary, truncates the version's input to 80 characters, and prints a line with the version, timestamp, and tokens; if input exists it prints a Python-style repr of that input on the next indented line. If more than five versions exist it prints a trailing line indicating how many older versions remain.

- Debug token table: when --debug-tokens is set the script prints a debug table for the latest version: it sorts the latest_tokens items by weight descending, prints the top 15 tokens with their weights, and marks tokens that are present in the query token set with " <- matched". If there are more than 15 tokens it prints a summary line indicating how many more tokens exist.

- Related chains computation: if args.related is nonzero and there are candidate_profiles, the program calls top_related_chains(latest_tokens, candidate_profiles, cid, cavm, k=args.related, min_score=args.related_threshold). That function returns up to k related chains scored by similarity to the target token vector. For each candidate profile it skips the target chain, requires the candidate have versions and tokens, and attempts to compute a similarity score using cavm.CosineLike(target_tokens, other_tokens). If cavm.CosineLike raises an exception, it falls back to a local cosine-like computation named fallback_cosine which computes the dot product of shared tokens divided by the product of L2 norms of both vectors (returning 0.0 if a denominator is zero). It only includes related chains whose similarity is >= min_score, then sorts the related list by score descending and returns the top k entries as tuples (score, cid, latest_version_number). For each related chain returned the main loop prints the related chain id, score, version number, and top tokens obtained by calling cavm.TopTokens on the related chain's latest tokens.

- Footer: after iterating all results the program prints a final divider, a line stating how many chains were found (len(results)), and if dormant chains were excluded it prints a reminder that dormant chains were excluded and that --all-chains can be used to include them.

- Helper utilities: iso_to_local attempts to parse an ISO timestamp with datetime.fromisoformat, assumes UTC when no timezone is present, and returns a localized "YYYY-MM-DD HH:MM:SS" string; it returns the original input string if parsing fails. age_human computes the elapsed seconds between now (UTC) and a provided datetime, normalizes naive datetimes to UTC, and returns a short human-readable age string.

- Error and early-exit behavior: if tokenization produces no tokens the program prints an error and exits with status 1. If cavm.Search returns an empty list the program prints "No matching chains found." and returns. If filtering out dormant chains removes all results it prints "No active chains match that query." and returns.

Overall, the program orchestrates tokenization, a search call to an external CAVM module, profile lookups, simple token-overlap statistics, related-chain similarity scoring (using either the CAVM-provided CosineLike or a local fallback), and formatted human-readable output describing each matching chain and optional details such as snippets, recent versions, debug token vectors, and similar chains.
