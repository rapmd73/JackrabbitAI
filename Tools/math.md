# Section 1 - Non-Technical Description

This program is a command-line calculator that reads a mathematical expression, either passed on the command line or entered interactively, evaluates the expression using standard arithmetic and parentheses, and prints the numeric result in a tidy format.

# Section 2 - Technical Analysis

The program defines a pipeline to parse, convert, evaluate, and format arithmetic expressions. It accepts expressions containing integers and decimal numbers, the four basic operators (+, -, *, /), and parentheses. The workflow is: tokenize the input string into individual tokens, convert the infix token sequence to postfix notation with the Shunting Yard algorithm, evaluate the postfix expression using a stack-based evaluator, and format the numeric result for output.

Tokenization (tokenize): The tokenize function scans the input string character by character, skipping whitespace. It recognizes numeric tokens by matching an optional integer part and optional decimal part via the regular expression '\d*\.?\d+' and appends matched numeric strings to the token list. It also recognizes single-character operator and parenthesis tokens from the set '+-*/()' and appends them. If an unexpected character is encountered, the function raises a ValueError. The function returns the ordered list of tokens (numbers as strings, operators, and parentheses).

Infix-to-postfix conversion (shunting_yard): The shunting_yard function takes the token list and converts it to postfix (Reverse Polish) notation. It uses a precedence mapping where '*' and '/' have precedence 2, and '+' and '-' have precedence 1. Numeric tokens (matched again by the same number regex) are appended directly to the output list. A left parenthesis '(' is pushed onto a stack; when a right parenthesis ')' is seen, operators are popped from the stack to the output until a matching '(' is found and removed. For operator tokens, the function pops operators from the stack to the output while the operator at the top of the stack has greater or equal precedence (and is not a '('), then pushes the current operator onto the stack. After processing all tokens, any remaining operators are popped to the output; if a '(' remains on the stack at that time, a ValueError is raised. The function returns the output list representing the postfix expression.

Postfix evaluation (evaluate_postfix): The evaluate_postfix function evaluates the postfix token list using a stack. It iterates tokens: when it encounters an operator, it pops two values from the stack (first popped becomes b, second becomes a), converts them to float, applies the operator (+, -, *, /) producing a numeric result, and pushes that result back on the stack. Division checks for a denominator whose absolute value is less than 1e-9 and raises ZeroDivisionError in that case. When the token is not an operator, it is converted to float and pushed on the stack. After processing all tokens the function expects exactly one value on the stack; otherwise it raises a ValueError. It returns the final numeric value as a float.

Result formatting (format_result): The format_result function receives a floating point value and returns a human-friendly string. If the float is mathematically an integer (value.is_integer() is True), it converts it to an int and returns the integer string. Otherwise it formats the float with six decimal places, then strips trailing zeros and an optional trailing decimal point, returning a compact decimal string without unnecessary trailing zeros.

Calculation pipeline (calculate): The calculate function composes the three main steps: tokenize the input expression string, convert the tokens to postfix with shunting_yard, and evaluate the postfix list with evaluate_postfix. It returns the numeric result produced by evaluate_postfix.

Command-line and interactive behavior (main): When invoked with command-line arguments, the program joins the arguments into a single expression string, replaces any lowercase 'x' characters with '*' (so 'x' acts as multiplication), runs the calculation pipeline, prints the formatted result to stdout, and exits with status 0 on success. If any exception occurs during calculation it prints an error message to stderr and exits with status 1.

When run without command-line arguments, the program enters an interactive loop. It prints two introductory lines describing the calculator and an example. It repeatedly prompts the user, reads a line of input, strips and lowercases it, and handles the following: if the user types 'exit' or 'quit' the program prints "Goodbye!" and stops; if the input is empty it prompts again; otherwise it runs the calculation pipeline on the entered expression and prints the formatted result prefixed by "= ". If an exception occurs during parsing or evaluation, the program prints an error message and continues prompting.

Program entry point: The script calls main() when executed as the main module, enabling both the single-expression command-line mode and the interactive REPL mode.
